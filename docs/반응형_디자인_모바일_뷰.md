# 반응형 디자인 및 모바일 뷰 가이드

## 목표

- 모바일 우선으로 설계하고, 큰 화면에서는 단계적으로 확장
- 동일한 정보 구조를 유지하면서 레이아웃만 재배치
- 터치/마우스 입력 모두에서 사용성 확보

## 기본 원칙

- Mobile-first: 기본 스타일은 모바일 기준, 필요할 때만 `sm:`, `md:` 이상으로 확장
- Fluid width: `w-full` + `max-w-*` 패턴으로 컨테이너 크기 제어
- 정보 우선순위: 좁은 화면에서 중요한 요소가 먼저 보이도록 재정렬
- 접근성: 터치 타겟 최소 44px, 텍스트 대비 유지
- 오버플로 방지: `min-w-0`, `truncate`, `break-words` 등으로 레이아웃 깨짐 방지

## Tailwind 브레이크포인트

- `sm` = 640px
- `md` = 768px
- `lg` = 1024px
- `xl` = 1280px
- `2xl` = 1536px

## 레이아웃 패턴

- 모바일 기준 + 화면 확대 시 단계적 확장

```tsx
<div className="mx-auto w-full max-w-[430px] sm:max-w-[520px] md:max-w-[720px] lg:max-w-[960px] bg-background shadow-sm">
  ...
</div>
```

- 카드/리스트 그리드

```tsx
<div className="grid grid-cols-1 gap-4 sm:grid-cols-2 lg:grid-cols-3">...</div>
```

- 타이포그래피 스케일

```tsx
<h2 className="text-lg sm:text-xl lg:text-2xl">Title</h2>
<p className="text-sm sm:text-base">Description</p>
```

## 모바일 뷰 기준

- 기준 폭: 360~430px 범위에서 정보가 깔끔하게 보이도록 설계
- 공간 단위: 4/8/12/16px 계열로 간격을 통일
- 이미지: `object-cover` + `aspect-*`로 비율 고정
- 고정 요소: 하단 네비/CTA는 `sticky` 또는 `fixed` 사용 시 safe area 고려

## PC/태블릿 확장

- 공통 레이아웃에서 모바일 기준 폭을 유지하되 `sm`, `md`, `lg`에서 확장
- 배경은 `bg-muted/40` 등으로 분리해 화면이 떠 보이도록 처리

## 체크리스트

- 360/430/768px에서 레이아웃 확인
- 긴 문자열/숫자/배지에서 줄바꿈 또는 잘림 확인
- `hover`만 의존한 UI는 `focus`/`active` 대체 동작 제공
