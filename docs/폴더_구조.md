# 프론트엔드 폴더 구조

## 구조 요약

```
src/
  assets/
  components/
  constants/
  hooks/
  layouts/
  lib/
  pages/
  stores/
  App.jsx
  App.css
  index.css
  main.jsx
```

## 폴더별 설명과 규칙

- `src/assets`
  - 정적 파일(SVG, 이미지, 폰트 등)
  - 파일명은 소문자/하이픈 권장
- `src/components`
  - 재사용 가능한 UI 컴포넌트
  - 파일명은 PascalCase, 한 파일 한 컴포넌트
- `src/constants`
  - 앱에서 공통으로 쓰는 상수
  - 런타임에 변하는 값은 넣지 않음
- `src/hooks`
  - 커스텀 훅 모음
  - 파일명은 `useXxx.js` 규칙
- `src/layouts`
  - 페이지의 공통 뼈대/레이아웃
  - `children`을 받는 컴포넌트 형태로 구성
- `src/lib`
  - 유틸리티/공용 모듈
  - `api.js`는 axios 인스턴스, `utils.js`는 `cn()` 제공
- `src/pages`
  - 라우트 단위 페이지
  - 기본 export로 페이지 컴포넌트 제공
- `src/stores`
  - 상태 관리(Zustand)
  - 파일명은 `useXxxStore.js` 규칙

## 루트 파일 역할

- `src/main.jsx`: 앱 시작점, 라우터/전역 스타일 연결
- `src/App.jsx`: 라우팅 정의 전용
- `src/index.css`: 전역 스타일과 Tailwind base 설정

## 확장 규칙

- shadcn/ui 컴포넌트는 필요 시 `src/components/ui`에 생성
- 페이지가 많아지면 `pages` 하위에 `features` 폴더를 추가
